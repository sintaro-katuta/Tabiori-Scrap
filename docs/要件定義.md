# 依頼：旅行計画＆アルバム共有アプリ「Tabiori-Scrap」要件定義書

3日間でのリリースを必達目標とし、以下の要件に基づき詳細な要件定義書を作成してください。

## 1. コンセプト
「旅行前から後までを一枚のタイムラインで繋ぐ、共同編集型のしおりアルバム」
- Before：チェックリストとマップURLで予定を立てる。
- After：その予定に紐づけて写真をアップし、スクラップブックを完成させる。

## 2. ターゲットユーザー
- 旅行の計画を友人と共有したい人。
- 旅の思い出を一つの場所で時系列に振り返りたい人。

## 3. 必須機能（MVP）
- **Auth**: Supabase AuthによるGoogleログイン。
- **Trip管理**: 旅行の新規作成、旅行ごとの固有URL発行。
- **Timeline**: 
    - 予定(PLAN)の追加（タイトル、時間、メモ、Google Map URL、チェック済フラグ）。
    - 写真(PHOTO)の投稿（Supabase Storageへのアップロード）。
- **Realtime**: 共同編集者が追加した内容がリロードなしで反映される機能。

## 4. UI/UX 要件
- **テーマ**: 温かみのあるデジタルスクラップブック。
- **カラー**: 背景はベージュ系（#fdf5e6）、アクセントは手書き感のあるブラウンや深緑。
- **レイアウト**: 縦型のタイムライン。写真はポラロイド風の枠線をつける。

## 出力物
- ユーザー体験フロー（User Journey）
- 各画面の機能詳細リスト
- 3日間のマイルストーン（いつまでに何を完了させるか）

## 5. 未確定事項・質問 (要確認)
以下の点について方針を決定し、追記してください。
- **共有範囲と権限 (Realtime/Auth)**:
    - Q: 「共同編集」の仕様詳細。URLを共有されたユーザーは即座に「編集権限」を持つか？
    - 案: MVPでは「URLを知っているログインユーザーは全員編集可能」とする（招待フローなし）。
- **デザイン詳細 (UI/UX)**:
    - Q: 「手書き風フォント」「紙の質感」の指定素材はあるか？
    - 案: 指定がなければ Google Fonts (例: Kaisei Decol, Yomogi) や CSS での表現を採用する。
- **Google Maps API**:
    - Q: 地図表示を行うか、単にリンクを保存するだけか？
    - 現状の要件: 「URL」とあるためリンク保存のみと想定。埋め込み表示が必要な場合はAPI Keyが必要。
