# 開発ガイドライン (Development Guidelines)

## Git・開発ワークフロー
本プロジェクトでは以下のプロセスを厳守して開発を行います。

1. **変更 (Changes)**
   - タスクごとにコードを変更する。

2. **コミット (Commit)**
   - メッセージは**20文字程度**で簡潔に記述する。
   - 例: `feat: ログイン機能実装`, `fix: 写真アップロード修正`

3. **プッシュ (Push)**
   - リモートリポジトリへプッシュする。

4. **PR作成 (Create Pull Request)**
   - mainブランチ（またはdevelopブランチ）に対してPRを作成する。
   - PRの概要には変更内容を明確に記載する。

5. **レビュー依頼 (Request Review)**
   - 担当者にレビューを依頼し、承認を得てからマージする（または承認フローをシミュレートする）。
